# Codebox 쉬운 설명서

> 이 문서는 "코드는 잘 모르지만, 이 프로젝트가 뭘 하는 건지 알고 싶다"는 분을 위한 안내입니다.

---

## 한 줄 요약

**브라우저 안에서 작은 창을 만들고, 키보드로 글자를 치면 화면에 보여 주는 연습용 프로그램**입니다.  
창은 드래그·닫기·최대화가 되고, 한글/영어 입력도 일부 지원합니다.

정답 문장 `"hello world"`를 맞춰 치는 **타자 연습**에 가깝게 만들어져 있습니다.

---

## 이 프로그램을 열면 무슨 일이 일어나나요?

```
1. index.html 이 브라우저에서 열림
2. index.js 가 자동으로 실행됨
3. 화면에 "창" 3개가 만들어짐
   ┌─────────────────────────────┐
   │  [닫기] [최대화]  ← 창 제목줄   │
   ├─────────────────────────────┤
   │                             │
   │   모니터 영역 (위쪽)          │
   │                             │
   ├─────────────────────────────┤
   │   입력 영역 (아래쪽)          │  ← 여기에 타이핑 결과 표시
   └─────────────────────────────┘
4. 키보드를 누르면 입력 영역 글자가 바뀜
```

HTML 파일 안에 버튼이나 입력칸이 미리 적혀 있지 **않습니다**.  
JavaScript가 실행되면서 화면을 **처음부터 직접 만들어** 넣습니다.

---

## 파일마다 하는 일 (비유로 설명)

### 🏠 `index.html` — 집의 문

- 브라우저가 가장 먼저 여는 파일
- CSS 꾸미기(`index.css`)와 JS 프로그램(`index.js`)을 불러옴
- 화면 내용은 거의 비어 있음 → JS가 채움

### 🎨 `index.css` — 인테리어

- 창 모양, 버튼 색, 닫기(✕) 표시 등 **겉모습** 담당
- `.window_bar` = 창 위쪽 초록색 제목줄
- `.intext` = 타이핑 결과가 보이는 글자 영역

### 🧠 `index.js` — 두뇌 (가장 중요)

하는 일을 크게 나누면:

| 하는 일 | 쉬운 설명 |
|--------|----------|
| 화면 만들기 | div 박스를 만들어서 창·입력칸·모니터 배치 |
| 창 기능 | 드래그, 닫기, 최대화/복원 버튼 연결 |
| 키보드 연결 | 키를 누를 때마다 글자 처리 |

**시작 함수:** 파일 맨 아래 `init()` 이 실행됩니다.

### ⌨️ `keyboardClass.js` — 타자 선생님

- 사용자가 친 글자를 기억하는 **세 개의 메모장**이 있음:
  - `input` — 그냥 친 글자 전부
  - `output` — 맞게 친 글자만
  - `target` — 맞춰야 할 정답 (`"hello world"`)
- 영어 모드: 정답과 한 글자씩 비교해서 맞으면 output에 추가
- 한글 모드: `hangulClass`에게 넘겨서 한글로 바꿈 (아직 완성 중)

### 🇰🇷 `hangulClass.js` — 한글 변환기

- 키보드 `r`, `k` 같은 **영문 키**를 받아서
- `ㄱ`, `ㅏ` 같은 **자모**로 바꾸고
- `가`, `각` 같은 **완성된 한글**로 이어 붙임
- 두벌식 키보드 기준 (한국에서 흔히 쓰는 배열)

### 📦 `sequent/` 폴더 — 이벤트 도구상자

"이벤트" = 클릭, 키 입력, 드래그 같은 **사용자 동작**.

보통은 `addEventListener`를 직접 쓰는데, 이 프로젝트는 `sequent`라는 **자체 도구**로 이벤트를 묶어서 관리합니다.

| 파일 | 역할 (쉬운 말) |
|------|---------------|
| `EventEmitter.js` | HTML 요소에 이벤트를 달아 두는 **콘센트** |
| `EventActionClass.js` | "이 키 누르면 이걸 해라" **한 가지 규칙** |
| `EventElementClass.js` | 규칙 여러 개를 **순서대로** 묶음 |
| `EventFlowClass.js` | "다음 단계로", "반복", "멈춰" 같은 **흐름 제어** |

예: 창 드래그 = ① 마우스 누름 → ② 움직임 반복 → ③ 마우스 뗌

## 키보드 단축키

| 키 | 동작 |
|----|------|
| 일반 글자 키 | 타이핑 (영어 모드 기본) |
| `Alt + N` | 한글 ↔ 영어 모드 전환 |
| `Ctrl + S` | 브라우저 기본 저장 막기 (아직 별도 기능 없음) |
| `Backspace` | 코드는 있지만 **아직 동작 안 함** |

## 영어로 "hello world" 치면 어떻게 되나요?

```
정답(target):  h e l l o   w o r l d
                ↓ ↓ ↓ ↓ ↓   ↓ ↓ ↓ ↓ ↓
사용자 입력:   h e l l o   w o r l d
                ↓
화면(output):  h e l l o   w o r l d  ← 맞춘 글자만 쌓임
```

한 글자라도 틀리면 그 글자는 output에 안 들어갑니다 (영어 모드 기준).

## 실행 방법

### 방법 1 — 그냥 파일 열기
`index.html` 을 브라우저로 더블클릭  
(일부 브라우저에서는 보안 때문에 안 될 수 있음)

### 방법 2 — 로컬 서버 (추천)
프로젝트 폴더에서:

```bash
npx serve .
```

터미널에 나온 주소(예: `http://localhost:3000`)를 브라우저에 입력

## 자주 묻는 질문

### Q. 왜 화면이 하얗게만 보이나요?
A. `index.js`가 실행되면 창이 생깁니다. JS가 꺼져 있거나, 모듈 로드 오류가 있으면 빈 화면일 수 있습니다. 브라우저 개발자 도구(F12) → Console 탭에서 에러를 확인해 보세요.

### Q. 한글은 되나요?
A. `Alt+N`으로 한글 모드로 바꿀 수 있지만, **아직 영어보다 덜 완성**되어 있습니다. 두벌식으로 `ㅎ`(g) `ㅏ`(k) 등을 치면 조합은 시도합니다.

### Q. 모니터(위쪽) 영역은 뭐에 쓰나요?
A. 지금은 **빈 박스**만 만들어 둔 상태입니다. 나중에 정답 표시, 점수, 진행률 등을 넣을 자리로 보입니다.

### Q. sequent는 꼭 필요한가요?
A. 이 프로젝트의 **핵심 실험 주제**입니다. "이벤트를 예쁘게 묶어서 재사용할 수 있을까?"를 테스트하기 위해 쓰고 있습니다. 단순 타자 연습만 하려면 sequent 없이도 만들 수 있지만, 지금 코드는 sequent에 많이 의존합니다.

### Q. `hello world`가 어디에 있나요?
A. `keyboardClass.js` 안에서 정답 문장이 `'hello world'`로 **하드코딩**되어 있습니다. 바꾸고 싶으면 이 값을 수정하면 됩니다.

## 마무리

이 프로젝트는 크게 **두 가지**를 동시에 연습하는 공간입니다.

1. **브라우저 UI 만들기** — JavaScript로 창, 버튼, 드래그 직접 구현  
2. **이벤트 시스템(sequent)** — 클릭·키입력·드래그를 깔끔하게 묶는 방법 연구  
3. **한글/영어 입력** — 타자 연습용 입력 처리

궁금한 파일이나 동작이 있으면 파일 이름을 알려 주시면, 그 부분만 더 풀어서 설명해 드릴 수 있습니다.
